Ha bármilyen kiegészítője vagy lokalizációja van, akkor a helyi és az addon fájlokat BOM nélkül kell konvertálni UTF-8-ra.

Létre kell hoznia egy MySQL adatbázist. Ezt a web-hosting vezérlőpultján vagy a phpMyAdmin-en keresztül teheti meg. Győződjön meg róla, hogy a MySQL hozzáférési adatai megtalálhatók, beleértve a gazdagépnevet, a felhasználónevet, a jelszót és az adatbázis nevét, mivel ezeket a beállítás során meg kell adni.

1. Töltse fel a fájlmappa tartalmát a webszerverére.

2. Ha a helyi kiszolgálón nem futtatod a PHP-Fusion-t, a legtöbb esetben a következő fájlokat és mappákat kell CHMOD-ra állítani 777-re:

  • administration/db_backups/
  • images/
  • images/imagelist.js
  • images/avatars/
  • robots.txt

A Core-val szállított infúziókhoz a következő mappákat kell CHMODed-re állítani 777-re

  • images/blog/images/
  • images/blog/images/thumbs/
  • images/blog/blog_cats/
  • downloads/
  • downloads/images/
  • downloads/submissions/
  • downloads/submissions/images/
  • forum/attachments/
  • images/news/images/
  • images/news/images/thumbs/
  • images/news/news_cats/
  • images/photoalbum/
  • images/photoalbum/submissions/

Megjegyzés:
Egyes kiszolgálók nem engedélyezik a CHMOD 777-et, ebben az esetben a CHMOD 755-et használhatja, ha a CHMOD 777 sikertelen.

3. Menjen a webhelyére, ahol a telepítés automatikusan elindul. Ha nem, akkor a telepítőt kézzel kell futtatnia a teljes webhely URL-címének beírásával, majd a setup.php . példával: http://www.yourdomain.com/setup.php.

4. Végezze el a telepítési folyamatot az összes képernyőn megjelenő utasítás követésével.

Frissítés v7.02.07 Ről

Mielőtt bármi más lenne, javasoljuk a fájlok és az adatbázis biztonsági mentését.
A PHP-Fusion 8 egy fontos frissítés a PHP-Fusion 7.02.07-ből.

Győződjön meg róla, hogy a tárhelyi környezet megfelel-e a verzió követelményeinek:
- PHP 5.6.40 vagy újabb
- MySQL 5.5.3 vagy újabb

This upgrade procedure can be very demanding depending on how much content your have.
If this procedure timeout or crash you need to manually disable the UTF-8 conversion script or if you can, raise the time of allowed PHP and SQL execution.
You disable the UTF-8 char conversion function by opening /administration/upgrade.php, line 135 change disabled = FALSE to disabled = TRUE.

PHP-Fusion 8.0 is compatible with version 7.02.07,
There have been a massive amount of alterations and additions to the Core systems, meaning that some Addons Mods, Infusions, Panels and Themes may need some alterations to work as intended with the system.

The upgrade will be in a few steps

You need to follow these instructions precisely.

Step 1

  • 1 · You must first upload all PHP-Fusion 8 files in the Upgrade_7.02.07_8.00.00. Replace all files when requested.
  • 2 · If your site is in English, navigate to your site's www.site.com and enter your Administration panel
    Go to System Admin > Upgrade and follow the instructions.
    If you upgrade from another language, make sure that you first translate the files in Upgrade_7.02.07_8.00.00/locale and replace the news ones in your v7 installation folders.
    When that is completed, Go to System Admin > Upgrade and follow the instructions.

Step 3
Once the database have been upgraded, copy all files and replace existing.

Step 4
Convert all your locales and addon files to UTF-8 without BOM.
Make sure that you also update your locales global.php and change charset from iso to utf-8 ( $locale['charset'] = "utf-8" ).
Notepad++ Can convert files if you do not find a good batch tool for this.

Step 5
If you do not run the site in English.
Synchronize you languages locale with the English set of locales before you start using the site to void errors.
You can simply open the English locale folder and compare to yours for missing files and changes.
We can recommend Notepad++ that have a compare files tool for example.
Open 1 English file and then your matching locale in window 2 press CTRL+ALT+C,repeat for each file.
We also have a compare tool available in the PHP-Fusion locales forums > Compare Tool.

Your site should now be fully upgraded and ready to run PHP-Fusion 8.

If you have any questions please feel free to ask for help in our Support Forums.

UPGRADE FROM v8.00.00

Before anything we strongly recommend that you backup your files and your database.

Make sure that your hosting environment meets the requirements for this version:
- PHP 5.6.39 or higher
- MySQL 5.5.3 or higher

The upgrade will be in a few steps

You need to follow these instructions precisely.

Step 1

  • 1 · You must first upload all PHP-Fusion 8 files in the Upgrade_8.00.00_8.00.19.
    Replace all files when requested.
  • 2 · Navigate to your site's www.site.com and enter your Administration panel
    When the file copy is completed, Go to System Admin > Upgrade and click upgrade.

Step 3
Once the database have been upgraded, copy all files and replace existing files.

Your site should now be fully upgraded and ready to run the next version of PHP-Fusion 8.

If you have any questions please feel free to ask for help in our Support Forums.

SECURITY TIPS
  • Ensure config.php is not writable (should be CHMODed to 644).
  • Always ensure your FTP and MySQL passwords are different.
  • Never allow forum,download attachments such as php, html, exe, or any type of text file.
  • Only use Addons that have been tested and approved as safe from our Marketplace.
  • Update your PHP-Fusion version when a new is released.
